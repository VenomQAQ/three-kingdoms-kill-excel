# WPS Excel 伪装 UI 设计

> 目标：在 PC 浏览器中 **一眼看上去正在使用 WPS 表格编辑数据**，实际进行三国杀对局。

## 1. 设计原则

1. **静态像 Excel** — 工具栏、公式栏、行列头、网格线、Sheet 标签与 WPS 高度一致
2. **动态藏游戏** — 游戏信息映射到「单元格内容」，避免明显游戏 UI（大图、卡牌动画）
3. **快捷键低调** — 操作尽量用键盘或右键菜单，减少 flashy 动效
4. **可切换「正经模式」** — 一键切到空白表格（老板键 / `Ctrl+Shift+H`）

## 5. 交互设计

### 5.1 进入游戏

1. 打开网页 → 默认显示假数据 Sheet1
2. 公式栏输入 `/join 48291037` → 切换到游戏 Sheet
3. 或点击状态栏隐藏入口

### 5.2 出牌流程

1. 点击手牌单元格（选中）
2. 点击目标玩家所在行（高亮）
3. `Enter` 确认 — 或右键「清除内容」伪装成删除数据

### 5.3 侧栏（操作区 & 聊天区）

#### 操作区（上半部分）
- **技能展示**：显示当前操作角色（由操控下拉框决定）的所有技能及详细说明
  - 格式：技能名称（绿色）+ 效果说明（灰色），每行一项
  - 当角色切换时实时更新

- **操作日志**：对局中的所有操作记录，按时间从上到下排列
  - 格式示例：
    ```
    【回合一】刘备 出牌阶段：
      1. 使用【杀】
      2. 打出【过河拆桥】
    ---- 刘备 出牌阶段结束，弃牌：【杀】、【闪】
    
    【回合二】关羽 出牌阶段：
      ...
    ```
  - 判定相关操作文字显示为红色加粗
  - 新增日志时自动滚动到底部

#### 聊天区（下半部分）
- **房间聊天**：所有玩家的聊天消息实时显示
  - 格式：昵称（绿色加粗）+ 冒号 + 消息内容
  - 新增消息时自动滚动到底部

#### 折叠/展开
- **折叠按钮**：侧栏标题栏右侧的 `◀` 按钮，点击后侧栏折叠为 32px 宽的展开条
- **展开按钮**：折叠状态下显示「展开」按钮，点击恢复为 360px 宽
- **目的**：让表格占满主区域，查看更多行数的玩家信息

### 5.4 老板键

- `Ctrl+Shift+H`：瞬间切到 Sheet1 假表格，暂停渲染游戏动画
- 再次按下恢复

### 5.5 引擎弹窗（GamePromptModal）

对局中有 `room.sandbox.prompt` 时，表格上方弹出 WPS 风格对话框（非全屏遮罩游戏感）：

| 弹窗标题 | 场景 |
|----------|------|
| 出牌确认 | 确认/取消打出 |
| 选择目标 | 杀、决斗、过河拆桥等选角色 |
| 响应 | 出闪/杀或承受效果 |
| 濒死救助 | 按座次询问是否使用【桃】救助；濒死者可用【桃】或【酒】自救 |
| 选择区域牌 | 过河拆桥/顺手牵羊：手牌显示「手牌 N」（匿名+乱序），装备明牌 |
| 弃牌阶段 | 选手牌弃置 |
| 改判 | 鬼才打出一张手牌，弹窗展示当前判定牌的生效说明 |
| 发动技能 | 准备阶段/摸牌前技能询问、观星排序、仁德/制衡多步 |

- 弹窗出现时客户端自动 `switchActor` 到 `prompt.playerId`
- 回合结束后自动切到当前回合角色
- 选区域牌弹窗支持**取消**
- 响应、改判、仁德、制衡等处理中弹窗支持点右上角 `×` 暂时收起，并从顶部“继续当前操作”提示条恢复
- 弃牌阶段关闭弹窗会返回出牌阶段；只有点击“确认弃牌”完成后才真正进入回合收尾
- 选中行和 hover 样式采用浅底深字，避免内容溢出或文字对比度不足

### 5.5 聊天

- 公式栏输入文字 Enter → 发送到房间
- 消息显示在公式栏下方窄条或「审阅」侧栏

## 6. 技术实现要点

```tsx
// 伪代码结构
<SpreadsheetShell>           {/* WPS 外壳 */}
  <MenuBar />
  <Toolbar />
  <FormulaBar onSubmit={handleChatOrCommand} />
  <Grid>
    {decoyData.map(...)}     {/* 前几行假数据 */}
    {gameRows.map(player => (
      <PlayerRow disguisedAs="salesRow" player={player} />
    ))}
  </Grid>
  <SheetTabs active={activeSheet} />
  <StatusBar roomCode={room.code} />
  {gamePrompt && (
    <GamePromptModal prompt={gamePrompt} … />  /* 对局弹窗 */
  )}
</SpreadsheetShell>
```

- 使用 **CSS Grid** 模拟单元格，不必上真实 spreadsheet 库（避免过重）
- 手牌、牌堆等敏感数据 **仅渲染在客户端**，从 WS 接收 filtered state
- 选将界面： disguised as 「数据验证」对话框

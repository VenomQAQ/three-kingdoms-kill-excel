# PRD · REQ-2026-005 · 玩家信息、聊天时间、战场铺满与连连看

- **版本**：v1
- **作者**：prd-author
- **状态**：①待三审消解
- **时间**：2026-07-05
- **前置**：REQ-2026-001、REQ-2026-003、REQ-2026-004

## 1. 背景与目标

### 1.1 现状问题

1. 当前界面仍有角色名前缀“界”，玩家反馈阅读噪音较大。
2. 聊天记录只有昵称与内容，没有发送时间，回看房间沟通时难以判断先后。
3. 玩家名称不可点击，无法查看昵称、战绩、金币等用户信息。
4. 顶部账号区域信息不足，无法直接看到等级与金币。
5. 战场右侧操作记录折叠后交互混乱，存在重复展开入口、侧栏空白和主表格未铺满问题。
6. 现有 Sheet 只有大厅/房间/伪装数据，缺少可消耗金币的轻量小游戏玩法。

### 1.2 目标

- 补齐玩家信息可见性，让昵称、等级、金币、战绩在合适位置可查。
- 提升聊天与战场操作记录的可读性。
- 继续强化 WPS 伪装壳，保证游戏区和新增连连看 Sheet 都以单元格铺满。
- 新增单人连连看玩法，形成“花费金币 -> 游戏挑战 -> 胜利奖励金币”的闭环，并为后续多人竞速预留扩展点。

### 1.3 成功指标

| 指标 | 目标 |
|---|---|
| 去“界”覆盖 | 主要 UI 中武将名前缀“界”展示残留为 0 |
| 聊天时间 | 大厅、房间、战场聊天消息 100% 展示发送时间 |
| 玩家资料 | 玩家名称点击后 1 次交互内打开资料弹窗 |
| 账号信息 | 登录态顶部可直接看到等级、昵称、金币 |
| 战场铺满 | 展开/折叠侧栏后，战场 Sheet 无明显空白断层，单元格延伸铺满视口 |
| 连连看闭环 | 金币不足不能开局；胜利奖励只发一次；失败不奖励 |

## 4. 交互与规则

### 4.1 武将名去“界”（R-UI-01）

- 展示层所有武将名统一去除开头的“界”。
- 包括：战场表格、选将、技能弹窗、对局日志、提示弹窗、装备/判定/手牌文本中的武将引用。
- 引擎内部、角色配置 id 可继续保留原始名称，避免破坏规则映射。
- 不得删除普通文本中的“世界”“边界”等非武将语境文字。

### 4.2 聊天时间（R-CHAT-01）

- 每条聊天消息展示发送时间。
- 当天消息格式：`HH:mm`。
- 非当天消息格式：`MM-DD HH:mm`。
- 大厅聊天、房间等待聊天、战场右侧聊天均适用。
- 历史消息使用服务端 `timestamp` 渲染，不使用客户端接收时间替代。

### 4.3 玩家信息（R-PROFILE-01 / R-PROFILE-02）

- 可点击入口：战场“用户”列、等待房间玩家昵称、聊天昵称、房间列表房主昵称。
- 弹窗展示：昵称、等级、金币、总局数、胜场、负场、胜率、最近更新时间。
- 自己与他人使用同一弹窗；他人资料不显示邮箱。
- 若战绩暂无数据，显示 0 值与空态，不阻塞弹窗打开。

### 4.4 右上角账号信息（R-ACCOUNT-01）

- 登录态显示：`Lv.{level} {nickname} · {coins}金币`。
- 未登录显示：`未登录`，点击仍打开登录入口。
- 收到 `user:walletChanged` 后即时刷新金币、经验、等级。
- 顶部信息应保持 WPS 标题栏风格，不做营销式卡片。

### 4.5 战场操作记录折叠与铺满（R-BATTLE-01 / R-BATTLE-02）

- 战场右侧侧栏包含“操作记录”和“聊天区”。
- 展开态：右侧宽度固定或在 320-380px 内响应式，主表格占剩余宽度。
- 折叠态：右侧只保留 32px 窄条与展开按钮，主表格自动扩展。
- 不允许同时出现多个展开按钮，不允许折叠后留出不可用空白。
- 主表格使用动态 filler 行/列铺满剩余区域，展开与折叠状态都要铺满。

### 4.6 连连看 Sheet（R-LLK-01~08）

**入口**

- 底部 Sheet 标签新增“连连看”。
- 未登录可浏览配置，但开始游戏需登录。

**基础规则**

- 点击两个未消除单元格。
- 两格内容相同且可通过不超过 2 次转弯路径连接，则消除。
- 全部图块消除且未超时为胜利；倒计时归零为失败。
- 重新开局视为新局，需要再次扣 5 金币。

**主题配置**

| 字段 | 说明 |
|---|---|
| themeId | 主题 id |
| name | 主题名 |
| items | 图块集合，含 `text` 与 `emoji` |
| similarGroups | 相似组，用于高难度提高辨认压力 |

初始主题建议：水果、三国武将、办公用品。

**难度配置**

| 难度 | 网格 | 种类 | 时间 | 入场费 | 胜利奖励 |
|---|---:|---:|---:|---:|---:|
| easy | 6x6 | 8 | 180s | 5 | 8 |
| normal | 8x8 | 14 | 150s | 5 | 15 |
| hard | 10x10 | 22 | 120s | 5 | 30 |

**展示模式**

- 支持 emoji 模式与纯文字模式切换。
- 同一局切换展示模式不改变棋盘数据。

**金币规则**

- 开局时服务端校验金币余额并扣除 5 金币。
- 胜利时服务端校验 session 状态并发放奖励。
- 同一 session 重复提交胜利不重复派奖。
- 失败、主动重开、离开 Sheet 均不返还入场费。

**竞速预留**

- session 结构预留：`mode: 'solo' | 'race'`、`roomId?`、`startedAt`、`finishedAt`、`difficultyId`、`themeId`、`result`。
- 本期只实现 `solo`。

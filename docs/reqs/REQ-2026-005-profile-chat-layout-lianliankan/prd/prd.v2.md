- **版本**：v2
- **作者**：prd-author
- **状态**：②三审问题已消解
- **时间**：2026-07-05
- **前置**：REQ-2026-001、REQ-2026-003、REQ-2026-004
- **SSOT 声明**：自本文档 v2 起，本轮需求以本文档为准；v1 与三审文档保留为历史证据。

## 1. 背景与目标

本轮迭代聚焦两条主线：

1. 优化现有三国杀 WPS 壳体验：去除武将名前缀“界”、聊天显示时间、玩家资料可查看、顶部展示等级昵称金币、战场操作记录折叠稳定并铺满单元格。
2. 新增“连连看”工作表：支持主题、emoji/文字展示、难度配置、倒计时、金币扣费与胜利奖励，并为后续多人竞速房预留结构。

### 1.1 成功指标

| 指标 | 目标 |
|---|---|
| 去“界”覆盖 | 主要 UI 武将名展示残留“界X”为 0 |
| 聊天时间 | 大厅、房间、战场聊天消息 100% 展示发送时间 |
| 玩家资料 | 点击玩家名 1 次交互内打开资料弹窗 |
| 顶部账号 | 登录态右上角展示 `Lv.{level} 昵称 · {coins}金币` |
| 铺满 | 战场展开/折叠、连连看 Sheet 均沿用 REQ-001 的 ≤3% 空白判据 |
| 金币闭环 | 金币不足不能开局；胜利只奖励一次；失败不奖励 |

## 4. 交互与规则

### 4.1 武将名去“界”（R-UI-01）

- 展示层所有武将名统一去除开头的“界”。
- 覆盖：战场表格、选将、技能弹窗、对局日志、提示弹窗、装备/判定/手牌文本中的武将引用。
- 引擎内部、角色配置 id 可继续保留原始名称，避免破坏规则映射。
- 不删除普通文本中的“世界”“边界”等非武将语境文字。

### 4.2 聊天时间（R-CHAT-01）

- 每条聊天消息展示发送时间。
- 当天消息格式：`HH:mm`。
- 非当天消息格式：`MM-DD HH:mm`。
- 大厅聊天、房间等待聊天、战场右侧聊天均适用。
- 历史消息使用服务端 `timestamp` 渲染，不使用客户端接收时间替代。

### 4.3 玩家信息（R-PROFILE-01 / R-PROFILE-02）

- 可点击入口：战场“用户”列、等待房间玩家昵称、聊天昵称、房间列表房主昵称。
- 弹窗展示：昵称、等级、金币、总局数、胜场、负场、胜率、最近更新时间。
- 他人资料本期允许展示金币；不展示邮箱。
- 若战绩暂无数据，返回并展示 0 值。
- 正式登录玩家的 `RoomPlayer.userId` 必须下发；虚拟玩家没有 userId 时，弹窗显示“虚拟角色”空态，不请求公开资料接口。

公开资料返回：

```json
{
  "userId": "usr_xxx",
  "nickname": "阿明",
  "level": 3,
  "coins": 128,
  "stats": { "total": 12, "wins": 5, "losses": 7, "winRate": 0.4167 },
  "updatedAt": 1783267200000,
  "_v": 1
}
```

### 4.4 右上角账号信息（R-ACCOUNT-01）

- 位置固定在 `TitleBar` 右侧、靠近“共享”按钮。
- 登录态显示：`Lv.{level} {nickname} · {coins}金币`。
- 未登录显示：`未登录`。
- 收到 `user:walletChanged` 后即时刷新金币、经验、等级。
- 保持 WPS 标题栏克制风格，不使用卡片。

### 4.5 战场操作记录折叠与铺满（R-BATTLE-01 / R-BATTLE-02）

- 战场右侧侧栏包含“操作记录”和“聊天区”。
- 展开态：右侧宽度 360px，主表格占剩余宽度。
- 折叠态：右侧只保留 32px 窄条与展开按钮，主表格自动扩展。
- 不允许同时出现多个展开按钮，不允许折叠后留出不可用空白。
- 主表格使用动态 filler 行/列铺满剩余区域。
- 验收沿用 REQ-001 的 3% 像素判据，覆盖 1440x900、1920x1080、2560x1440。

### 4.6 连连看 Sheet（R-LLK-01~08）

**入口**

- 底部 Sheet 标签新增“连连看”。
- 未登录可浏览配置，但开始游戏需登录。

**基础规则**

- 点击两个未消除单元格。
- 两格内容相同且可通过不超过 2 次转弯路径连接，则消除。
- 全部图块消除且未超时为胜利；倒计时归零为失败。
- 重新开局视为新局，需要再次扣 5 金币。

**主题配置**

| 字段 | 说明 |
|---|---|
| themeId | 主题 id |
| name | 主题名 |
| items | 图块集合，含 `text` 与 `emoji` |
| similarGroups | 相似组，用于高难度提高辨认压力 |

初始主题：水果、三国武将、办公用品。

**难度配置**

| 难度 | 网格 | 种类 | 时间 | 入场费 | 胜利奖励 |
|---|---:|---:|---:|---:|---:|
| easy | 6x6 | 8 | 180s | 5 | 8 |
| normal | 8x8 | 14 | 150s | 5 | 15 |
| hard | 10x10 | 22 | 120s | 5 | 30 |

**展示模式**

- 支持 emoji 模式与纯文字模式切换。
- 同一局切换展示模式不改变棋盘数据。

**金币与结算**

- `POST /api/lianliankan/sessions` 开局时服务端校验金币余额并扣除 5 金币。
- 后端创建 session，返回棋盘、倒计时、奖励、sessionId。
- 前端负责连线交互与本局棋盘消除状态。
- `POST /api/lianliankan/sessions/:id/finish` 提交结果；后端校验 session、超时、幂等状态。
- 胜利奖励按难度发放；失败、主动重开、离开 Sheet 不返还入场费。
- 同一 session 重复提交胜利不重复奖励。

**竞速预留**

- session 结构预留：`mode: 'solo' | 'race'`、`roomId?`、`startedAt`、`finishedAt`、`difficultyId`、`themeId`、`result`。
- 本期只实现 `solo`。

## 5. 错误码

| 错误码 | 场景 | 文案建议 |
|---|---|---|
| E_WALLET_INSUFFICIENT_COINS | 金币不足 5，不能开始连连看 | 金币不足，无法开始本局 |
| E_LLK_SESSION_NOT_FOUND | 连连看 session 不存在 | 本局已失效 |
| E_LLK_SESSION_SETTLED | session 已结算 | 本局已结算 |
| E_LLK_SESSION_EXPIRED | 超过通关时间提交胜利 | 已超时，挑战失败 |
| E_LLK_INVALID_CONFIG | 难度或主题不存在 | 连连看配置不存在 |

## 1. 总体判断

需求可测，但连连看金币结算、战场铺满和“界”字去除需要可重复的测试夹具。建议在设计阶段明确配置、错误码、接口返回与 UI 断言点。

## 2. 测试关注点

| 范围 | 关注点 |
|---|---|
| 去“界” | 不能误删普通词；所有武将展示入口覆盖 |
| 聊天时间 | 当天/跨天格式、历史消息、滚动到底 |
| 玩家资料 | 自己/他人/无战绩/断网失败态 |
| 账号栏 | 登录/未登录、钱包变更后刷新 |
| 战场侧栏 | 多次折叠、不同分辨率、展开后聊天输入可用 |
| 连连看 | 难度配置、路径规则、倒计时、金币不足、重复领奖、失败不返钱 |

## 1. 逐条可测性检查

| 需求点 | 可测 | 备注 |
|---|---|---|
| R-1 UI 风格锁定 | 半可测 | "不允许随意变更"是流程约束，需要辅以"截图 diff 基线"作为回归判据 |
| R-2 单元格填满 | 可测 | 需要"空白像素占比 ≤ 3%" 的量化脚本；QA 会用 puppeteer / playwright 截图 + canvas 像素扫描 |
| R-3 注册 | 可测 | 用例覆盖：邮箱正则、密码强度、重复注册、并发注册 |
| R-4 登录/登出 | 可测 | 覆盖：正确/错误密码、连续错次限流、登出后 session 失效、断线重连保持登录 |
| R-5 改密码 | 可测 | 覆盖：原密码错、新密码不合规、成功后旧 token 失效 |
| R-6 版本目录 & 切换 | 可测 | 覆盖：切换后房间列表过滤、切换事件广播到同账号多 socket |
| R-7 房间容量 | 可测 | 覆盖：满员再加入被拒 + 专属错误码；换版本后新房容量正确 |
| R-8 大厅聊天 | 可测 | 覆盖：大厅频道广播、进入房间后不收大厅、聊天节流、消息长度截断 |
| R-9 未登录兜底 | 可测 | 覆盖：直接调 socket / http 都返回 E_UNAUTHORIZED |

## 2. 需要 PRD 补齐的判据

1. **R-1 视觉基线**：需要"当前 UI 截图作为基线"，QA 会保存到 `qa/baseline/*.png`，PRD 应授权保存。
2. **R-2 空白像素测量方法**：3% 的计算方式（是"非表格背景色像素 / 总像素"，还是"完全空白（RGB 全等）像素"）需要 PRD 明确。**建议**：非表格背景色像素占比。
3. **R-8 大厅聊天节流阈值**：PRD 里"1 条/秒/账号"来自后端评审员建议，PRD 需正式采纳或改。
4. **R-4 断线重连的时限**：session TTL / 允许多久重连仍能保持登录，PRD 未定。

## 3. 三国杀语境专项检查

- 本需求**不改战斗规则**，出牌流程 / 技能生效路径不动 → 无需回归全部技能牌用例（qa-test-designer 增加"冒烟集"即可）。
- 版本切换**只影响 room.version 与容量**，引擎路径不变 → 判定牌种子接口不受影响。

## 4. 通过条件

1. 补齐 §2 的四项判据。
2. 授权 QA 建立"当前 UI 视觉基线"。
3. 明确大厅聊天历史保留策略（用例设计需要知道）。

## 5. 判定

`pass-with-conditions`：全部需求点可测，但 R-1/R-2/R-8 需要 PRD 把数值/方法说清楚才不至于用例含糊。

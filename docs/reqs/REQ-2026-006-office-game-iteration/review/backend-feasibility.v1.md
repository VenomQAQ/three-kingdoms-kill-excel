# 后端可行性评审 v1

结论：通过。

## 评审点

| 项 | 结论 | 说明 |
| --- | --- | --- |
| 分游戏战绩 | pass | `PlayerPublicProfile` 可扩展为 `statsByGame`，当前无历史数据时返回 0 值 |
| 连连看配置 | pass | 配置常量即可调整难度与主题，不涉及迁移 |
| 大富翁房间类型 | pass | `Room` 可增加 `gameType`，`room:create/list/start` 根据类型分支 |
| 大富翁状态 | pass | 首期可在 `Room.monopoly` 中维护轻量状态，内存房间与现有架构一致 |
| 免费游玩 | pass | 创建/开始逻辑不触达钱包扣费 |

## 风险与处理

- R-BE-1：`versionId` 当前承担三国杀版本过滤，和游戏类型混用会让大富翁消失。处理：新增 `gameType` 过滤字段，三国杀继续使用 `versionId`。
- R-BE-2：大富翁完整世界版规则过大。处理：本期提供基础世界城市路线、掷骰、购买、过路费和回合轮转。
- R-BE-3：公开资料契约变更需兼容老前端。处理：保留 `stats` 字段，同时新增 `statsByGame`。

最终意见：通过，进入设计。
